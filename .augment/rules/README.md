# 项目规则说明

本目录包含 AI 助手在开发过程中必须遵守的规则。

## 📋 规则列表

### 1. file_creation_policy.md - 文件创建和组织规则
**核心原则**：未经用户明确要求，禁止创建任何新文件

- ❌ 禁止主动创建文档、测试文件、脚本等
- ✅ 仅在用户明确要求时才创建文件
- 📁 定义了项目文件组织规范
- 🔄 提供了文件移动和整理指南

### 2. manual_testing.md - 手动测试规则
**核心原则**：不要自动启动服务，提供测试指南

- ❌ 不要使用 `launch-process` 启动前端或后端服务
- ✅ 提供清晰的手动测试指南
- 📝 列出测试要点和预期效果

### 3. frontend_structure.md - 前端结构规则
**核心原则**：遵循前端项目结构规范

- 定义前端目录结构
- 组件命名和组织规范
- 样式和资源管理规则

### 4. workflow_rules.md - 工作流开发规则
**核心原则**：工作流功能开发规范

- 工作流节点开发规范
- API 设计规范
- 数据流转规范

## 🎯 使用说明

### 对于 AI 助手

在开发过程中，必须：
1. **优先查看相关规则**：开始任务前先查看相关规则文档
2. **严格遵守规则**：不得违反规则中的禁止项
3. **询问用户**：遇到规则未覆盖的情况时询问用户

### 对于开发者

如需添加或修改规则：
1. 在本目录创建或编辑 `.md` 文件
2. 使用清晰的标题和结构
3. 明确标注 ✅ 允许的行为和 ❌ 禁止的行为
4. 提供具体示例

## 📝 规则优先级

1. **用户明确指令** > 所有规则
2. **file_creation_policy.md** > 其他规则（关于文件创建）
3. **项目架构规范** (docs/ARCHITECTURE_GUIDE.md) > 具体实现规则

## 🔄 规则更新

- 规则应该随着项目发展不断完善
- 发现规则冲突时，优先遵守更具体的规则
- 规则应该简洁明了，避免过度复杂

## ⚠️ 重要提醒

这些规则是为了：
- 保持项目结构整洁
- 避免创建不必要的文件
- 确保开发流程一致性
- 提高代码质量

**不是为了限制开发，而是为了更好地协作！**
